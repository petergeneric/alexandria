# C FFI Architecture

## Overview

The macOS Swift app communicates with `alexandria-core` through a C FFI layer generated by `cbindgen`. The Rust library is compiled as a static library (`staticlib`) and linked into the Xcode project.

## Planned API Surface

```c
// Opaque handle to the search engine
typedef struct AlexandriaEngine AlexandriaEngine;

// Search result returned over FFI
typedef struct {
    const char *url;
    const char *title;
    const char *content_snippet;
    const char *domain;
    float score;
} AlexandriaSearchResult;

typedef struct {
    const AlexandriaSearchResult *results;
    uintptr_t count;
} AlexandriaSearchResults;

// Lifecycle
AlexandriaEngine *alexandria_engine_open(const char *index_path);
void alexandria_engine_close(AlexandriaEngine *engine);

// Search
AlexandriaSearchResults *alexandria_engine_search(
    const AlexandriaEngine *engine,
    const char *query,
    uintptr_t limit,
    uintptr_t offset
);
void alexandria_search_results_free(AlexandriaSearchResults *results);
```

## Memory Management Rules

1. **Rust owns all allocations**: Every pointer returned by Rust must be freed by calling the corresponding `_free` function
2. **Strings**: Returned as `const char *` (null-terminated). Caller must NOT free them directly; they are freed when the parent struct is freed
3. **Engine handle**: Opened with `_open`, closed with `_close`. Caller must not use the handle after closing
4. **Thread safety**: The engine handle is `Send + Sync`. Multiple threads can search concurrently

## cbindgen Setup

`crates/alexandria-core/build.rs` runs cbindgen to generate `alexandria.h`:

```rust
fn main() {
    let crate_dir = std::env::var("CARGO_MANIFEST_DIR").unwrap();
    cbindgen::Builder::new()
        .with_crate(crate_dir)
        .with_language(cbindgen::Language::C)
        .generate()
        .expect("Unable to generate bindings")
        .write_to_file("target/alexandria.h");
}
```

## Swift Integration

The generated `alexandria.h` is added as a bridging header in the Xcode project. Swift calls the C functions directly:

```swift
let engine = alexandria_engine_open(indexPath)
defer { alexandria_engine_close(engine) }

let results = alexandria_engine_search(engine, query, 10, 0)
defer { alexandria_search_results_free(results) }

for i in 0..<results.pointee.count {
    let result = results.pointee.results[i]
    let title = String(cString: result.title)
    // ...
}
```
